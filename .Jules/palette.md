## 2025-02-12 - Sticky Header Keyboard Navigation Bypass
**Learning:** This app features a persistent, sticky global header. Keyboard-only users are forced to tab through the site identity and navigation links on every route change before reaching the primary page content. This is a common but frustrating accessibility barrier.
**Action:** Implemented a 'Skip to main content' link. By defining a `skipToContentPolicy` in `app/constants/policy.ts`, the link remains visually hidden (`sr-only`) until focused (`focus-visible:not-sr-only`). Crucially, the target `<main>` element must receive `tabIndex={-1}` to accept programmatic focus and `focus:outline-none` to avoid drawing a distracting focus ring around the entire page content. This pattern should be standard for all layouts with persistent top navigation.
