## 2024-04-25 - Sticky Header Accessibility and Skip Links
**Learning:** The application uses a sticky global header, which means keyboard users would have to tab through the navigation on every page load unless a bypass mechanism is provided. However, the target container must handle focus correctly without showing an unwanted focus ring when receiving programmatic focus from the skip link.
**Action:** Implemented a visually hidden "Skip to main content" link at the top of the document (styled via `skipToContentPolicy`). The target `<main>` element was made programmatically focusable with `id="main-content"` and `tabIndex={-1}`, and explicitly styled with `focus:outline-none` to prevent visual disruption while ensuring robust keyboard accessibility. Always use this pattern for anchor link targets that are not inherently interactive.
